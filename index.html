예: Meta + Google / 카카오 + 네이버 — 멀티 채널 집행 시 채널 간 성과 비교와 어트리뷰션이 중요해집니다

✓

수동 → 핵심 → 파워 페르소나 전환 요인을 찾아 더 많은 유저를 파워유저로 만듭니다
✓
파워유저 / 핵심유저 / 수동유저를 행동 데이터로 구분해서 본 적 있다
단순 가입일/금액 기준이 아닌, '얼마나 자주 핵심 이벤트를 수행하는가'로 나누는 것이 핵심
✓
파워유저가 다른 유저에 비해 '더 자주 하는 행동'이 무엇인지 알고 있다
예: 파워유저는 첫 1주일 내 '팔로우' 기능을 5회 이상 사용 — 이 행동을 신규유저에게 유도하면 됩니다

리텐션 라이프사이클 — 유저는 지금 어느 단계에 있나요?
신규 / 현재 / 복귀 / 휴면 유저 분포로 제품 성장의 건강도를 진단합니다
✓
신규 유저 / 현재 유저 / 복귀 유저 / 휴면 유저의 비율을 현재 파악하고 있다
신규: 첫 사용 주기 | 현재: 지난 주기도 이번 주기도 활성 | 복귀: 휴면 후 재방문 | 휴면: 이번 주기 비활성
✓
액티브 유저 중 '현재 유저(반복 방문자)' 비율이 70% 이상인지 알고 있다
현재 유저 비율이 낮으면 신규 유입에만 의존하는 구조 — 리텐션 커브가 계속 내려갑니다
현재 우리의 D7 리텐션율은 대략 얼마인가요? 직접 입력
✓
(신규 + 복귀 유저) > 휴면 유저 — 즉 잃는 유저보다 얻는 유저가 더 많다
Pulse Ratio = (신규+복귀) / 휴면 > 1 이면 진정한 성장, < 1 이면 역성장 중
